# Usability Checklist

## Task Efficiency

- [ ] Primary task can be completed in minimum steps
- [ ] Default values are sensible for most users
- [ ] Search is available where finding items is part of workflow
- [ ] Keyboard shortcuts for power users (documented)
- [ ] Bulk operations where applicable
- [ ] Most frequent action has most visual weight
- [ ] Form auto-focus on first field (when appropriate)

## Navigation

- [ ] User always knows current location
- [ ] Back navigation works as expected
- [ ] Breadcrumbs for multi-level navigation
- [ ] Navigation items use recognizable labels (not brand terms)
- [ ] Active navigation state is visually distinct
- [ ] No dead ends (every screen has a next action)
- [ ] Deep linking works (URL represents state)

## Feedback

- [ ] Every action has immediate feedback (≤ 100ms) or loading indicator
- [ ] Error messages explain what happened and how to fix it
- [ ] Success confirmations are subtle (not blocking)
- [ ] Loading states show skeleton not spinner
- [ ] Optimistic updates for predictable operations
- [ ] Undo available for destructive actions (or confirmation)

## Content

- [ ] Primary action label is a verb ("Save", not "Okay")
- [ ] Buttons say what they do ("Delete account", not "Submit")
- [ ] Microcopy is clear and concise
- [ ] No jargon without explanation
- [ ] Dates and times use relative format ("2 hours ago") or full format consistently
- [ ] Numbers formatted consistently (currency, decimals, thousands separator)

## Error Prevention

- [ ] Destructive actions have confirmation or undo
- [ ] Form data preserved on validation error
- [ ] Auto-save for long forms
- [ ] Session timeout warning with save option
- [ ] Network status indicator for online-dependent features
- [ ] Validation is real-time (inline after field blur)
- [ ] Confirmation step before irreversible actions

## Onboarding

- [ ] First-run experience guides user to first meaningful action
- [ ] Empty states offer a clear next action
- [ ] Feature discovery is contextual (not a tour)
- [ ] Help/documentation is accessible within context
- [ ] Tutorials are skippable
